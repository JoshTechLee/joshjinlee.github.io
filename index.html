Projects

  • Client-Server-HotSpot

    A bare-bones TCP client-server hotspot created as a final project for my Computer Networking class. Each connection had to recreate the TCP three-way handshake to send and receive real data. The user made a request to the server and the server retrieved real data from the URL byte-by-byte. Users were simulated using multiple threads (limited by design to concurrent connections).

    • Python

  • Quibble

    For my Software Quality Assurance final project, we were tasked upon creating a ticket kiosk that tracked customer data. The kiosk had to regulate and restrict users depending on whether they were customers or admins. Customers were only allowed to purchase and refund tickets, while admins were allowed to set prices, create events, etc. At the end of each day, all purchases for that day are saved in a unique file, as well as the number of tickets left. To test the authenticity, various white-box and black-box test scripts were made such as unit tests and functional tests.

    • Python
    • Batch
